# 任务模板

这份文档给出 3 类高频任务的默认模板。目标不是把流程写死，而是减少每次开工前重新组织任务的成本。

## 1. Bugfix 模板

适用场景：线上问题、回归问题、局部行为异常。

建议记录：

- 问题现象
- 影响范围
- 复现步骤
- 预期行为
- 根因
- 修复策略
- 验证方式

默认动作：

1. 先复现，再修复。
2. 优先补回归测试或最小验证。
3. 如果问题源于约束缺失，把规则补到最近的 `AGENTS.md`、测试或校验脚本。

## 2. 新功能模板

适用场景：新增页面、接口、交互能力、后台模块。

建议记录：

- 目标
- 用户价值
- 不做什么
- 影响目录
- 新增接口 / 数据结构
- 验收标准

默认动作：

1. 先写清楚边界和验收标准。
2. 实现时保持改动聚焦，不顺手扩散重构。
3. 如果引入新目录、新模式或新流程，同步补文档导航。

## 3. 重构模板

适用场景：目录整理、模块边界收敛、命名统一、基础设施调整。

建议记录：

- 当前问题
- 重构目标
- 不改变的行为
- 影响范围
- 风险点
- 验证方式

默认动作：

1. 先定义“不变项”，避免重构变成隐式改需求。
2. 先做机械迁移，再做结构收敛。
3. 重构完成后，优先更新 `AGENTS.md`、README 和一致性检查。

## 通用收尾清单

- 文档是否仍准确
- 是否需要新增测试
- 是否需要新增 consistency check / lint / CI 规则
- 是否应把经验沉淀到 `docs/delivery-workflow.md`
