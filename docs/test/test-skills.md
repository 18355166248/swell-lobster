现在清楚了，你的 SKILLS/ 下有 6 个技能：

┌──────────────────┬────────────────────────────────┐
│ Skill │ 触发方式 │
├──────────────────┼────────────────────────────────┤
│ docx │ 文档操作，通过描述任务自动触发 │
├──────────────────┼────────────────────────────────┤
│ emoji-expression │ 轻松氛围时自动插入 emoji │
├──────────────────┼────────────────────────────────┤
│ pdf │ PDF 操作，通过描述任务自动触发 │
├──────────────────┼────────────────────────────────┤
│ pptx │ PPT 操作，通过描述任务自动触发 │
├──────────────────┼────────────────────────────────┤
│ skill-creator │ 创建/优化技能 │
├──────────────────┼────────────────────────────────┤
│ xlsx │ 表格操作，通过描述任务自动触发 │
└──────────────────┴────────────────────────────────┘

这些 skill 是通过 description 字段做语义匹配自动路由的，不需要 / 命令。以下是验证话术：

---

docx
帮我创建一个 Word 文档，内容是"Hello World"，保存为 docx

pdf
帮我把这段文字生成一个 PDF 文件：这是一份测试文档

pptx
帮我创建一个 PPT，主题是"项目介绍"，包含标题页和一页内容页

xlsx
帮我创建一个 Excel 表格，包含姓名、年龄、分数三列，填入3行示例数据

skill-creator
帮我创建一个新技能，功能是将中文翻译成英文

emoji-expression
今天终于把这个 bug 修好了，太开心了！

---

注意：docx/pdf/pptx/xlsx 这四个依赖 run_script 工具执行 Python/Node 脚本，如果你的桌面端没有接入 run_script
工具，这四个会触发但无法实际执行。先用 emoji-expression 和 skill-creator 验证路由是否正常。
