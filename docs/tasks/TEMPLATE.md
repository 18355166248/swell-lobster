# 任务标题

- 为什么要做这件事

## 影响范围

- 影响的目录

## 方案

- 核心思路
- 关键约束
- 风险点

## 验收标准

- 完成后如何判断任务已达标

## 验证

- `npm run verify:docs`

- 其他专项验证

- 是否需要新增测试 / consistency check / CI 规则
