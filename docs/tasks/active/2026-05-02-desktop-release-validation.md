# 桌面安装包与升级链路验证

## 背景

- 桌面 sidecar、自检、导出和日志主链路已经补齐，但仍缺少真实桌面环境验收记录
- 最近的顶部栏重启与默认聊天页体验调整，需要在开发态与打包态分别确认行为

## 目标

- 完成一轮桌面应用真实环境验证
- 形成安装、启动、升级、导出与日志排障的验收清单

## 不做什么

- 不扩展新的 Tauri 插件
- 不在本任务内新增阶段 10 的 IM 或 Agent 协作能力
- 不重做桌面 UI 结构

## 影响范围

- 桌面壳：`apps/desktop/`
- Web UI 桌面交互：`apps/web-ui/`
- 验证与状态文档：`docs/PROJECT_STATUS.md`、`docs/roadmap.md`

## 方案

- 分开发态与打包态执行验证，覆盖启动、重启、sidecar、导出和日志
- 重点验证默认进入聊天页、顶部重启按钮、文件导出打开与日志落盘
- 将验证结果沉淀为可复用清单，后续用于发布前回归

## 当前产出

- 已新增 `docs/desktop-validation-checklist.md` 作为发布前人工验收模板
- 待执行真实桌面环境验证并回填验证记录

## 验收标准

- `npm run build:desktop` 成功
- 安装包首次启动正常，后端健康检查通过
- 开发态点击顶部“重启”不会白屏；打包态点击“重启”可重新拉起应用
- 导出文件、本地打开、日志路径与 sidecar 行为均可验证

## 验证

- `npm run verify`
- `npm run build:desktop`
- 人工验证安装、启动、重启、导出、日志与升级链路

## 沉淀项

- 需要把稳定的桌面验收项补到发布前检查清单
- 如验证发现稳定性缺口，再新开专项任务，不直接把问题留在本任务中
