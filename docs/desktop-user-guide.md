# SwellLobster Desktop 使用手册

本文面向已经安装或准备使用桌面版的终端用户，重点说明首次配置、核心功能、各页面用途，以及桌面端特有的日志、重启和文件打开能力。  
如果你要排查 sidecar、开发态启动链路或桌面打包问题，请配合阅读 [runtime-guide.md](runtime-guide.md) 与 [desktop-env-config.md](desktop-env-config.md)。

## 适用范围

- 适用于 SwellLobster Desktop 打包版用户
- 适用于希望在本地长期使用个人 AI 助手的内部同事
- 文中页面名称与当前桌面导航保持一致：聊天、消息通道、技能、MCP、计划任务、记忆管理、日记、状态面板、Token 统计，以及配置页

## 开始前你需要知道

### 这是什么

SwellLobster Desktop 是一个本地桌面 AI 助手。桌面应用本身是宿主，真正提供聊天、工具、记忆、计划任务和消息通道能力的是内置后端服务。

你可以把它理解为：

```text
桌面应用
  -> 应用界面
  -> 本地后端服务
  -> 本地数据目录、日志、导出文件
```

### 首次启动时会发生什么

首次打开桌面应用后，程序会尝试启动内置后端，并在顶部栏显示当前连接状态。

- 如果顶部显示“运行中”，说明桌面端已经可以正常使用
- 如果顶部显示“未连接”，优先去“状态面板”查看运行信息与日志入口

### 数据保存在哪里

桌面端会在本地保存运行数据、配置和导出文件。

- 默认导出目录：`~/Documents/SwellLobster/outputs`
- 全局环境变量文件：
  - macOS：`~/.swell-lobster/.env`
  - Windows：`%USERPROFILE%\.swell-lobster\.env`
- 日志文件：
  - macOS：`~/Library/Logs/ai.swell.lobster/tide-lobster.log`
  - Windows：`%AppData%/ai.swell.lobster/logs/tide-lobster.log`

## 首次使用流程

建议按下面顺序完成第一次配置。

### 1. 确认服务已启动

打开应用后先看顶部栏：

- 状态为“运行中”时，可以继续配置和使用
- 状态为“未连接”时，先进入“状态面板”查看：
  - 运行模式
  - 配置文件路径
  - 数据目录
  - 当前进程 ID
  - 日志文件入口

### 2. 配置 LLM 端点

进入“配置” -> “LLM 端点”。

这是最重要的第一步。没有可用端点时，聊天页无法正常调用模型。

你通常需要完成这些操作：

1. 点击“添加端点”
2. 选择服务商
3. 填写 API 地址、模型名和 API Key
4. 保存配置
5. 如有需要，点击“应用并重启”

这个页面还支持：

- 配置多个模型端点
- 设置优先级
- 配置故障转移目标
- 配置语音识别端点（STT）
- 设置“提示词编译模型”

适用场景：

- 你需要接入 OpenAI 兼容接口
- 你要区分快模型、便宜模型和高质量模型
- 你想让主模型失败时自动切到备用模型

### 3. 按需配置全局 `.env`

如果你所在网络环境需要代理，或者你希望统一配置 API Key、搜索服务、输出目录，推荐使用全局 `.env` 文件。

常见变量包括：

- `HTTP_PROXY`
- `HTTPS_PROXY`
- `ALL_PROXY`
- `NO_PROXY`
- `SWELL_OUTPUT_DIR`
- `SWELL_PYTHON_BIN`

完整写法见 [desktop-env-config.md](desktop-env-config.md)。

推荐场景：

- 模型接口必须经过本地代理访问
- 你需要自定义导出目录
- 你需要让桌面重装后仍沿用同一套环境配置

### 4. 检查语言、主题与重启入口

顶部栏提供几个常用入口：

- 运行状态
- 已加载端点数量
- 主题切换
- 中文 / English 语言切换
- 重启

说明：

- 打包桌面版中，“重启”会尝试重启内置后端
- 开发态中，“重启”更接近刷新窗口

## 界面结构

桌面端的主要入口都在左侧导航。

### 聊天

主工作区，用于与 AI 助手对话。

支持：

- 新建对话
- 多会话切换
- 会话重命名
- 会话置顶
- 历史搜索
- 删除会话
- 导出 Markdown / JSON
- 选择人设
- 选择 Agent 模板
- 图片上传
- 文件上传
- 语音输入按钮

### 消息通道

用于管理 Telegram、钉钉等 IM 通道，让助手通过外部消息渠道工作。钉钉当前使用 Stream 模式，需要在配置中填写 `Client ID / Client Secret`。

### 技能

用于查看和管理技能，支持内置技能与用户技能。

### MCP

用于接入外部工具服务器，扩展助手可调用的工具能力。

### 计划任务

用于创建定时任务，让助手按日、周、月或自定义 Cron 自动执行。

### 记忆管理

用于查看、筛选、搜索、编辑和删除长期记忆。

### 日记

用于记录日记、查看心情与时间线，以及和记忆联动。

### 状态面板

用于查看当前桌面端的服务状态、运行模式、日志入口和后端进程信息。

### Token 统计

用于查看模型调用消耗、使用趋势和成本估算。

### 配置

配置区包含四类页面：

- LLM 端点
- IM 通道
- 身份配置
- 高级配置

## 核心使用方式

## 聊天与会话管理

聊天页是大多数用户每天都会使用的主入口。

### 新建对话

点击“新建对话”后，可以直接开始，也可以先选择 Agent 模板。

适合的做法：

- 日常聊天直接创建空白会话
- 固定类型工作优先选择模板，例如写作、研究、代码辅助

### 发送文本、图片和文件

聊天框支持多种输入方式：

- 文本消息
- 图片附件
- 文件附件
- 语音输入

已知行为：

- 当前模型不支持视觉能力时，图片会被提示移除
- 文件类附件可作为上下文交给模型处理

推荐场景：

- 上传截图让模型解释界面或报错
- 上传 Markdown、TXT、PDF 做总结或提炼
- 使用语音输入快速记录任务想法

### 会话整理

每个会话支持这些管理动作：

- 重命名
- 置顶 / 取消置顶
- 批量选择
- 删除
- 导出 Markdown
- 导出 JSON

常见用法：

- 用置顶保留长期项目对话
- 用搜索快速回到旧结论
- 用 Markdown 导出沉淀可读版本
- 用 JSON 导出做备份或结构化处理

### 人设与模板

聊天会话支持切换人设，也支持在创建时选择 Agent 模板。

这两者的区别：

- 人设偏风格与身份设定
- 模板偏任务类型与工具组合

如果你想让同一个模型在不同场景里表现不同，可以优先使用这两个入口，而不是每次都重写很长的指令。

## 功能页面说明

### 消息通道

“消息通道”页面用于管理外部 IM 接入。

你可以在这里：

- 新增消息通道
- 编辑通道配置
- 启动 / 停止通道
- 删除通道
- 查看通道状态
- 对 Telegram 等需要配对的通道做授权管理

适合的使用方式：

- 给自己配置 Telegram Bot，让助手在手机上也可访问
- 给团队机器人配置钉钉 Stream 通道

前置条件：

- 相关 Token、Webhook、白名单等环境项已准备好
- 如果连接失败，优先回到“状态面板”看日志

### 技能

“技能”页面用于查看当前可用技能，以及管理技能的启用状态和来源。

适合的使用场景：

- 想了解助手默认会哪些能力
- 想验证某个技能文件是否已被加载
- 想区分内置技能和用户自定义技能

如果你只是使用者，不需要频繁修改这里；它更像是能力清单和调试入口。

### MCP

“MCP” 页面用于给助手接入外部工具服务器。

你可以在这里：

- 查看已安装服务器
- 启用 / 停用服务器
- 查看服务器状态
- 查看该服务器暴露的工具
- 从市场安装 MCP
- 手动新增自定义 MCP

适合的使用场景：

- 想让助手接入网页搜索、第三方 API 或本地工具
- 想排查某个工具为什么在聊天里不可用

如果某个 MCP 启动超时，先检查：

- 网络代理是否可用
- npm registry 是否正确
- 对应环境变量是否已经写入全局 `.env`

### 计划任务

“计划任务”用于定时执行提示词任务。

你可以在这里：

- 创建每日、每周、每月任务
- 使用自定义 Cron 表达式
- 指定某个端点执行任务
- 手动立即运行任务
- 查看最近执行记录

适合的使用场景：

- 每天早上生成工作摘要
- 每周定时整理待办
- 固定时间调用搜索或分析任务

建议：

- 先用手动执行验证任务提示词
- 再打开定时执行，避免错误任务反复触发

### 记忆管理

“记忆管理”用于维护长期记忆。

你可以在这里：

- 按类型筛选记忆
- 搜索记忆
- 新增记忆
- 编辑记忆内容
- 删除单条记忆
- 清空全部记忆

记忆类型包括：

- 事实
- 偏好
- 事件
- 规则

适合的使用方式：

- 把稳定偏好写成记忆，而不是每次都重复告诉助手
- 清理无效或过时的长期记忆，减少干扰

### 日记

“日记”页面用于个人记录与回顾。

当前能力重点包括：

- 写日记
- 记录心情
- 查看时间线
- 查看与记忆的关联
- 在更新后触发记忆提取

适合的使用场景：

- 每日复盘
- 跟踪长期目标和情绪变化
- 把生活记录和助手记忆打通

### 状态面板

“状态面板”是桌面端最重要的排障入口。

你可以在这里直接看到：

- 服务状态
- 运行模式
- 当前环境文件路径
- 项目根目录
- 数据目录
- 后端进程 ID
- 可执行文件路径

桌面端还提供两个直接操作：

- 重启后端
- 查看日志

建议在这些情况下优先打开“状态面板”：

- 顶部显示“未连接”
- 模型列表加载失败
- IM / MCP / 计划任务表现异常
- 你不确定当前用的是哪份 `.env`

### Token 统计

“Token 统计”用于查看用量与趋势。

你可以在这里关注：

- 总体消耗趋势
- 各模型使用情况
- 估算成本

前提是相关端点已填写价格信息，这样统计页才能更准确地估算费用。

## 配置页面说明

### LLM 端点

这是桌面端最核心的配置页。

适合维护：

- 主聊天模型
- 备用模型
- 语音识别模型
- 提示词编译模型

推荐做法：

1. 至少准备一个稳定主端点
2. 为关键端点配置 API Key 环境变量名
3. 按需要补充 fallback 端点
4. 保存后刷新模型列表或应用并重启

### IM 通道

“IM 通道”配置页当前更适合做 IM 相关配置确认与跳转入口。

你可以在这里：

- 查看与 Telegram、钉钉等通道相关的环境变量是否已经被读到
- 快速跳转到“消息通道”页管理 Bot
- 快速跳转到“高级配置”页统一编辑 `.env`

它适合配合“消息通道”页一起使用：

- “消息通道”负责具体通道实例管理
- “IM 通道”负责确认 IM 相关环境变量是否存在
- “高级配置”负责实际编辑桌面端全局环境变量

### 身份配置

“身份配置”用于管理 persona、提示资产等与助手身份相关的内容。

适合的使用场景：

- 调整默认人格风格
- 查看当前身份资产文件
- 在多种工作模式之间切换

如果你希望助手在“陪伴型”和“工作型”之间切换，这个页面比反复修改系统提示更合适。

### 高级配置

“高级配置”用于较少改动但影响全局行为的设置。

当前重点包括：

- Agent 名称
- 隐藏模块
- Embedding 配置
- 语义检索阈值
- 搜索 Provider
- 搜索 API Key
- 某些全局密钥项
- 全局环境变量编辑

适合的使用场景：

- 你想把某些模块从侧边栏隐藏掉
- 你要启用向量记忆检索
- 你要切换 Brave、Tavily 或 DuckDuckGo 搜索
- 你要统一维护 IM、MCP、代理等 `.env` 配置

## 桌面端特有能力

这些能力是桌面版相对浏览器版最有价值的部分。

### 打开日志

在“状态面板”中点击“查看日志”，应用会调用系统默认方式打开日志文件。

适合用于：

- 给研发同学提供错误日志
- 自己确认模型请求失败、sidecar 启动失败或 MCP 超时原因

### 重启后端

桌面端支持从顶部栏或“状态面板”直接重启后端。

适合的使用场景：

- 改了全局 `.env` 后希望重新加载
- 模型列表长时间无法刷新
- 某些 MCP / IM 通道初始化状态异常

### 使用系统默认程序打开导出文件

聊天导出成功后，文件会落到输出目录。桌面端会尽可能调用系统默认程序打开相关文件。

如果文件已生成但打不开，通常是系统文件关联问题，不一定是导出失败。

### 全局 `.env`

桌面端会优先读取用户主目录下的全局 `.env`，这意味着：

- 应用重装后配置仍可保留
- 不需要修改安装目录
- 更适合保存代理和 API Key 相关环境项

## 常见问题

### 顶部显示“未连接”

先进入“状态面板”，确认：

- 是否拿到了运行模式和进程信息
- 能否打开日志
- `.env` 路径是否符合预期

如果日志里有明显报错，再按 [runtime-guide.md](runtime-guide.md) 继续排查。

### 模型列表刷新失败

优先检查三件事：

1. LLM 端点中的 API 地址、模型名和 API Key 是否正确
2. 是否需要代理
3. 全局 `.env` 是否已经生效

如果使用代理，重点看 [desktop-env-config.md](desktop-env-config.md)。

### MCP 安装或启动超时

优先检查：

- 当前网络代理
- npm registry
- 该 MCP 需要的环境变量是否已写入

这类问题通常不是聊天页本身的问题，而是外部工具进程没有成功启动。

### 导出文件找不到

先检查：

- 默认目录 `~/Documents/SwellLobster/outputs`
- 是否配置了 `SWELL_OUTPUT_DIR`
- 日志里是否有导出失败记录

### 修改配置后没有生效

依次尝试：

1. 在相关页面重新保存
2. 使用顶部栏“重启”或“状态面板”中的“重启后端”
3. 重新打开应用

### 不想看到某些模块

进入“配置” -> “高级配置”，使用“隐藏模块”功能即可。

适合隐藏的页面通常包括：

- 不使用的 IM
- 不使用的 MCP
- 暂时不需要的计划任务、记忆管理或 Token 统计

## 推荐使用顺序

如果你是第一次正式使用桌面端，建议按下面顺序熟悉：

1. 先完成“LLM 端点”配置
2. 回到聊天页完成第一次对话
3. 尝试上传一张图片或一个文件
4. 试一次会话导出
5. 去“状态面板”确认日志入口与环境路径
6. 再按需使用记忆、日记、计划任务、MCP 和消息通道
