# Docs — 指南

## 目录职责

这个目录存放应该进入仓库的正式产品、工程与流程文档。

- 架构说明
- Prompt 或工作流文档
- 会影响实现的团队参考资料

不要把没有长期工程价值的临时笔记直接堆到 `docs/`。

## 规则

- 如果文档改变了实现预期，要从最近的 `AGENTS.md` 或 README 链接过去。
- 优先保存解释系统行为的稳定文档，而不是一次性的会议纪要。
- 文件名保持可读，可持续演进的话题用子目录归类。
